🧠 知识层

Convention B：Tcl = LiDAR→Camera

关键发现：Tcl 矩阵含义为 LiDAR→Camera（P_cam = Tcl @ P_lidar），而非 Camera→LiDAR。

验证方法：LiDAR 环在图像中应水平（平行宽度方向）。Convention A 导致 cam0/cam2 环垂直，Convention B 全部水平。

着色变换链：

P_cam = Tcl @ inv(T_world_lidar) @ P_world
其中 T_world_lidar = T_world_body @ Til
T_world_body = img_pos 位置 + 四元数旋转
已验证
标定外参 着色管线

PLY 点云坐标系

关键发现：PLY section 中的点云为世界坐标系，非本地 LiDAR 坐标系。

证据：PLY 质心跟随设备位置移动（质心-设备距离始终 <5m），反变换到本地后质心在原点附近。

MANIFOLD 设备在存储时已将本地扫描变换到世界坐标系。

已确认
点云数据 着色管线

.lx Section 格式

Header 48B（float[1:7]=位姿 tx/ty/tz/qx/qy/qz/qw，float[8]=section编号）+ Point Count 4B + Points N×16B。

与 img_pos.txt 行号一一对应（7,631 sections）。

已解析

坐标系变换树

  World (PLY 点云坐标系)
    │
    │  T_world_lidar = T_world_body @ Til
    │  (img_pos 四元数+位置 → T_world_body)
    │
    ▼
  LiDAR
    │
    │  Tcl (LiDAR→Camera, Convention B)
    │
    ▼
  Cam{0,1,2}
