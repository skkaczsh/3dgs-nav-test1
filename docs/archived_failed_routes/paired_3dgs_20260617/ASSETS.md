# 技术资产沉淀总表

> 本文件汇总项目开发过程中验证过的模式、踩过的坑、可复用的代码片段。
> 按类别组织，便于未来检索和复用。
> 
> **维护规则：**
> - 每次调试完成后 → 追加"踩坑记录"
> - 每次任务完成后 → 追加"可复用代码"或"设计模式"
> - 每次架构选择后 → 追加"ADR"
> - 发现有用工具/命令 → 追加"工具备忘"
> - 定期回顾，合并重复项，标记过时项
