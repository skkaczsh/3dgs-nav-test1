🧩 核心模块

📦 SplatLoader

状态: Active

加载 .splat 二进制文件，解析 Gaussian 参数（position, scale, color, rotation）。

关键能力

  • 支持 gsplat .splat 格式（32 bytes/Gaussian）
  • 支持 .ply 格式（Open3D fallback）
  • 空间范围过滤（移除 NaN/Inf 异常点）
  • 坐标系变换：PCD(Z-up) → Splat(Y-up)

关键参数

  • position: float32 米（非毫米！）
  • scale: float32 ÷ 100 → 分米级
  • opacity: logit 存储，避免双重 sigmoid
  • rotation: uint8 [0,255] → float32 quaternion
踩坑：双重 Sigmoid → GaussianRenderer 坐标系风险

🎬 GaussianRenderer

状态: Active

基于 gsplat 的多视图渲染器，输出 RGB/Depth/Alpha 三通道图像。

关键能力

  • gsplat.rasterization 光栅化
  • COLMAP 相机加载 + 虚拟相机生成
  • 相机 180° Y轴旋转（修复朝向问题）
  • 分块渲染避免显存溢出

渲染参数

  • near_plane: -400.0, far_plane: 500.0
  • 分辨率: 800×640（保持 K 矩阵一致）
  • 深度: max-alpha 深度（最前面 Gaussian）
踩坑：深度归一化 踩坑：相机朝向 ← SplatLoader → SAM2

🎭 SAM2MaskGenerator

状态: Active

Segment Anything Model 2 全自动分割，生成逐像素 mask。

关键能力

  • 32×32 点阵均匀采样（1024 点/图）
  • 基于 IoU + Stability Score 过滤
  • 贪心去重叠算法
  • GPU 并发加速

模型配置

  • hiera_tiny / hiera_large
  • pred_iou_thresh: 0.7
  • stability_score_thresh: 0.92
  • min_mask_area: 100-500 px
← GaussianRenderer → VLM 迭代优化模式

🤖 VLM Client

状态: Active

视觉语言模型客户端，支持本地 Qwen2-VL 和远程 HTTP API。

多阶段处理

  • ANALYZE: 检查 mask 质量，发现可合并区域
  • CLASSIFY: 为每个 mask 区域标注语义类别
  • VERIFY: 验证并修正分类结果

标签体系

  • 18 类: wall, floor, ceiling, table, chair, ...
  • 中文 → 英文映射（LABEL_MAP）
  • 异步并发 16 workers
← SAM2 → VoxelGrid 踩坑：JSON 解析

📐 VoxelGrid

状态: Active

点云体素化，为语义融合提供空间索引。

关键能力

  • Open3D 体素化引擎
  • 可配置体素大小（默认 0.05m）
  • 体素中心计算 + 点索引映射

输出

  • Voxel 对象列表（center, point_indices, votes）
  • voxel_map: (i,j,k) → voxel_id
→ SemanticFusion 资产：体素化模式

🔗 SemanticFusion

状态: Open

语义融合引擎：体素中心投影到多视图，读取 mask 标签，加权投票。

融合流程

  • 体素中心 → 投影到每个视图
  • Alpha Gate: α > 0.5
  • Depth Gate: |z_render - z_voxel| < threshold
  • 读取 mask 标签 → 加权投票
  • 置信度 > 0.5 才采纳

输出

  • 每个 Voxel: final_label + confidence
  • assign_labels_to_points(): 体素标签 → 点标签
← VoxelGrid ← VLM ⚠️ Unknown Labels Bug 坐标系风险

⚠️ 风险区域

🧭 坐标系对齐风险

状态: Open

PCD (Z-up) 和 Splat (Y-up) 坐标系不同，且存在 X 轴反转。

已知问题

  • PCD X 方向与 Splat X 方向相反
  • COLMAP 相机朝向需 180° Y轴旋转
  • unit_scale 参数易混淆（cm → m）

缓解措施

  • TIMELINE.md 记录了完整变换公式
  • 渲染前检查相机 forward 方向

Unknown Labels Bug

严重度: Critical

E2E 测试后所有体素标签为 unknown，语义融合完全失败。

症状

  • Voxel label distribution: unknown 100%
  • Labeled PCD: 0/24M points

可能根因

  • mask 坐标 → 体素坐标投影失败
  • VLM 标签未写入体素 votes
  • Alpha/Depth Gate 过于严格
SemanticFusion 坐标系风险

双重 Sigmoid

严重度: Fixed

SplatLoader + get_opacity_sigmoid 双重应用 sigmoid → alpha=0.0016。

修复

  • 移除 SplatLoader 中的 sigmoid
  • 使用 logit 存储 opacity
SplatLoader

深度归一化

严重度: Fixed

gsplat 返回归一化深度 [0,1]，非线性米制深度。

根因

  • gsplat 内部: d_norm = (far - Z) / (far - near)
  • 输出 ~0.999 是正确的物理深度
GaussianRenderer

相机朝向错误

严重度: Fixed

COLMAP 相机朝向 SPLAT 正 X，但 PCD 在负 X 区域。

修复

  • 180° Y轴旋转: R_y = [[-1,0,0],[0,1,0],[0,0,-1]]
  • 验证 forward 方向指向 PCD
坐标系风险 GaussianRenderer

VLM JSON 解析

严重度: Fixed

VLM API 返回截断 JSON 或 markdown fences，导致解析失败。

修复

  • 支持 markdown code fences 移除
  • 截断 JSON 修复（补全 ]}）
  • 非贪婪正则匹配
