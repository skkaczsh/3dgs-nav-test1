## Goal

Validate whether `GroundingDINO + SAM2` is a viable fine-object route for
`railing / pipe / HVAC outdoor unit` on the existing rooftop scan imagery.

This is a small-sample 2D evaluation only. It does not change the main point
cloud route yet.

## Environment finding

Two different remote environments were checked:

- `vlm_seg` fails for GroundingDINO because its `transformers` version is too
  new for the local GroundingDINO integration (`BertModel.get_head_mask`
  missing).
- `conceptseg-r1` can load GroundingDINO and SAM2 successfully. GroundingDINO
  runs with CPU fallback because the custom CUDA op is not available, while
  SAM2 still runs on CUDA.

Therefore the practical small-sample path is:

- GroundingDINO on CPU
- SAM2 on CUDA

Prompt style:

- One large caption containing all candidate terms:

Observed issue:

  cross-class detections.

### B. Narrowed per-sample prompt

Output:

- `/Users/skkac/Work/SCAN/new_route/experiments/fine_object_grounded_small_eval/outputs_focus`

Prompt style:

- Each sample uses a smaller caption based on its `focus`.

- Better than the full mixed prompt.
- Railing recall improved, but cross-class coupling remained because
  `railing/equipment` still shared one caption.

Focus match summary:

- `railing`: `7 / 11`
- `pipe`: `2 / 2`

Focus match summary:

## Main conclusion

`GroundingDINO + SAM2` is now **operational** on the small sample set, and the
detector route is **meaningfully better for fine-object recall** than using one
fully mixed caption.

The key positive signal is:

- grouping prompts by focus raised `railing` presence recall from `4 / 11` to
  `11 / 11`

This is enough to justify continuing this route.

## Remaining problem

The remaining failure is not detector absence. It is **over-large detections**,
especially for `equipment/HVAC` and sometimes `metal fence`.

Grouped-run median/max mask areas:

- `railing`: median `35,795`, max `1,952,741`
- `pipe`: median `24,612`, max `105,479`
- `equipment`: median `42,146`, max `1,499,001`
- `hvac`: median `66,211`, max `1,153,894`

Interpretation:

- `railing` recall is now good enough to continue.
- `pipe` looks plausible and bounded.
- `equipment/HVAC` frequently expands into large background regions and cannot
  be accepted directly.

## Practical next step

Do not return to the old `SAM2 full-image + VLM per-mask classification` route
for fine objects.

The next correct step is to keep this grouped detector route and add guards:

1. box or mask area upper bounds for `equipment/HVAC`
2. elongated-shape and width/height heuristics for `railing`
3. sky/depth/surface rejection before point projection
4. point-cloud-side confirmation after projection
